Display GitHub repository metadata in a compact card that matches Nexus UI styling. `RepoCard` is a presentational component that accepts pre-fetched data, while `fetchGitHubRepo` is available when you want to load GitHub metadata on the server first.

## Usage

```tsx keepBackground
import { RepoCard, fetchGitHubRepo } from "@/components/nexus-ui/repo-card";
```

```tsx keepBackground
const data = await fetchGitHubRepo("shadcn-ui", "ui");

<RepoCard owner="shadcn-ui" repo="ui" data={data} />
```

Use `fetchGitHubRepo` in a server component, route handler, or loader when you want cached GitHub metadata. Pass `data` to `RepoCard` to render the card without any async work during client rendering.

## Notes

- `RepoCard` is safe to use in both client and server components because it no longer performs async fetches during render.
- Use `fetchGitHubRepo` when you want cached GitHub API data before rendering the card.
- The optional `GITHUB_TOKEN` environment variable raises GitHub API rate limits for `fetchGitHubRepo`.
- Archived repositories render a warning badge; forked repositories render a fork badge.
- Unknown languages fall back to a neutral dot color.
